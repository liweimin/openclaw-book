# OpenClaw 标准配置指南 (命令行向导版)

如果你是第一次使用 OpenClaw，或者需要重新配置你的大模型 API、通信插件（如飞书、微信等），我们强烈推荐使用**自带的交互式命令行向导**，而不是手动修改配置文件或使用尚在完善中的 Web 界面。

## 💡 为什么推荐使用命令行配置？
1. **防止手误**:手动修改 `openclaw.json` 极易因为少一个逗号或括号导致整个程序崩溃。向导会自动帮你校验并生成正确的格式。
2. **步骤无遗漏**: 向导涵盖了从“选择大模型”到“配置网络端口”、“绑定聊天软件”等8个关键的标准化步骤。
3. **安全稳定**: 相比于目前仍在迭代、可能会遇到按钮失效等 Bug 的 WebUI，命令行的底层逻辑是最成熟和稳定的。

---

## 🚀 标准配置步骤

打开你电脑上的终端工具（CMD 或 PowerShell），输入以下命令启动按向导：

- **如果你使用的是正式版 (npm 安装)**:
  ```powershell
  openclaw config
  ```
- **如果你使用的是源码版**:
  ```powershell
  cd d:\code\anzhuang\openclaw
  pnpm openclaw config --allow-unconfigured
  ```

按回车后，向导会一步一步引导你完成以下核心步骤（你可以用键盘上下方向键选择，回车确认）：

1. **Workspace (工作区)**: 设置工作区名称，它是你所有聊天记忆隔离的基础。
2. **Model (模型配置)**: 选择你的大语言模型服务商（如 OpenAI、Anthropic、智谱、Kimi 等），并输入你的 API Key。这一步非常关键。
3. **Web tools (网络工具)**: 配置搜索引擎（如用于联网搜索的 API）和网页抓取能力。
4. **Gateway (网关及网络)**: 设置 OpenClaw 运行的网络端口、绑定地址以及登录鉴权（设置你的访问密码）。
5. **Daemon (后台服务)**: 配置是否让 OpenClaw 作为后台常驻服务运行（小白用户一般跳过即可）。
6. **Channels (通道接入)**: 将 OpenClaw 连接到微信、飞书、钉钉、Telegram 等聊天软件，实现“替身”功能。
7. **Skills (扩展技能)**: 激活各种内置的工作流套件。
8. **Health check (健康检查)**: 最后一步，程序会自动帮你检查网络连接、API 密钥是否有效。

## 🛠️ 高级技巧：精确修改某一项参数

如果你只想修改一个非常小的配置（比如换一个 API 秘钥），不想从头过一遍向导，你可以使用精确的 `set` 命令。

比如，你想临时换绑一个其他的模型，你可以这样输入：

```powershell
# 语法: openclaw config set <设置路径> <设置值>

# 例如修改后台运行端口
openclaw config set gateway.port 18789

# 例如修改飞书插件的参数
openclaw config set channels.feishu.appId "cli_a8xxx999"
```

这种单行修改的方式不仅绝对安全（自带格式校验），而且**完全可以实现知识复用**。不论你是在源码测试环境还是生产环境，修改命令完全一样！
