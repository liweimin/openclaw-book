# 术语表与核验说明

## 1. 为什么要有这一页

如果这本书后面要继续整理成 PDF、EPUB 或可对外发布的电子书，那么术语和事实口径必须稳定。

这一页承担两个作用：

1. 统一术语
2. 标明关键事实是如何核验的

## 2. 术语表

### OpenClaw

指整套个人 AI 助手系统与框架，而不是单独某个模型或某个网页界面。

### Gateway

指 OpenClaw 的长期运行中枢。  
它负责连接、状态、会话、事件、控制面和部分安全边界。

### Dashboard / Control UI

指浏览器中的控制界面。  
它是入口，不是整个系统本体。

### Agent

指真正执行推理、任务编排和工具调用的运行实体。

### Agent Runtime

指 Agent 实际运行的机制和执行流程。

### Tool / Tools

指 Agent 能调用的执行能力，例如文件、命令、浏览器、消息、设备等。

### Skill / Skills

指给 Agent 的可复用任务套路、行为说明和流程约束。

### Memory

指长期记忆层。  
它和当前会话上下文有关，但不等于单轮上下文。

### Channel / Channels

指各类消息入口，如 WhatsApp、Telegram、Discord、Feishu 等。

### Pairing

指显式批准访问的安全流程。  
既可用于私信配对，也可用于节点设备配对。

### Node / Nodes

指加入 OpenClaw 网络的额外设备能力端或节点端。

### Binding / Bindings

指 Agent 与具体渠道入口的路由绑定关系。  
例如：把特定飞书机器人账号绑定到 `weekly-assistant` Agent。

### Workspace

指 Agent 专属的工作区目录。  
存放其独有的 `AGENTS.md`、`MEMORY.md`、Skills、记忆数据等运行时文件。

### Cron

指 OpenClaw 的定时触发机制。  
允许 Agent 在没有人发送消息时，按固定时间或频率主动执行任务（如发周报）。

### Hook

指在 Tool 调用前后或生命周期关键节点介入的机制。  
常用于安全审批、参数改写或执行拦截。

### ClawHub

指 OpenClaw 的公共技能中心或注册表。  
可以通过 `clawhub search` 和 `clawhub install` 获取他人写好的 Skill。

### Session / sessionKey / runId

- **Session**：一次持续的会话上下文。
- **sessionKey**：代表这类会话归属的长线逻辑分类（如来自飞书某个私聊）。
- **runId**：某次具体 AI 推理和工具执行周期的唯一标识码。

### web_search / web_fetch / browser

搜索相关的三个层次内置工具：
- **web_search**：用于寻找信息源（搜索引擎 API）。
- **web_fetch**：用于快速读取静态网页的正文内容。
- **browser**：用于启动浏览器获取需要渲染、交互的安全验证页面内容。

### QMD

指进阶记忆检索使用的特定查询路由和后端。

### Compaction

指因为对话历史太长而自动触发的上下文压缩机制。

### dmScope

指某个通道下私聊会话隔离的作用域范围。

## 3. 本书中最重要的核验结论

以下内容在写书时都被视为“关键事实”，优先参考官方源码和官网文档：

### OpenClaw 的官方定位

官方定位是 `personal AI assistant`。

### Windows 推荐路线

官方推荐通过 WSL2 使用 OpenClaw，而不是原生 Windows。

### 当前源码 Node 要求

当前官方源码中的 Node 要求是：

```text
>=22.12.0
```

### onboarding 的关键地位

对第一次安装来说，`openclaw onboard --install-daemon` 是最关键的主入口之一。

### Dashboard 的角色

`openclaw dashboard` 是最快完成第一次本地验证的路径之一。

### Gateway 默认本地入口

当前默认本地入口仍围绕 `127.0.0.1:18789` 展开。

## 5. 为什么要特别强调“关键事实核验”

因为 OpenClaw 这类系统最容易过时的，不是概念性语言，而是这些内容：

- 平台支持状态
- 安装方式
- Node 版本要求
- CLI 命令名和参数
- 默认端口
- 安全默认值
- 当前源码链路位置

如果这些内容写错，整本书就会对外误导读者。  
所以这本书在这些地方，优先以官方当前版本为准，而不是只沿用旧整理文档里的说法。

- 术语表

- 版本说明

这样后面更新版本时，会更容易维护。
