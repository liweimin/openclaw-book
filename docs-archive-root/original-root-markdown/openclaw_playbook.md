# 🎮 OpenClaw 新手调教与进阶玩法指南

恭喜你成功配置好了一个基于 OpenAI Codex 大脑且直连飞书的超级本地助理！

然而你发现它的第一反应竟然是：“我不能联网搜索”。别急，这不是它的错，而是 **API 机制** 的区别。

---

## 🔍 第一问：为什么 ChatGPT 能搜，它却说不行？

**原理解析：**
你平时在网页端使用的 ChatGPT Plus，是自带了微软 Bing 搜索插件的“精装房”。而我们通过 OAuth/API 接入的 Codex 大脑，是原汁原味、没有外挂的“纯脑组织”。它不知道今天是几号，也不能自己打开浏览器。

**OpenClaw 是怎么解决这个问题的？**
OpenClaw 的强大之处在于它能给这个“纯脑”安装各种工具（Plugins）。我们刚才在配置里帮你启用了 `browser` (浏览器) 和 `code-search` (代码搜索) 插件。

**那为什么刚才还是报错（超时/缺 Key）？**
1. **`browser` 抓取服务超时**：这是因为 OpenClaw 在幕后悄悄使用了一个叫 `Playwright` 的自动化无头浏览器。你第一次触发它时，它可能正在后台疯狂下载几十百兆的 Chromium 浏览器内核，导致回复超时（这也是国内网络环境常遇到的坑）。
2. **`web_search` 缺少 API**：单纯的“搜索新闻”通常需要一个搜索引擎的接口（比如 Brave Search），而我们暂时没有配置。

### 🔧 解决方案

**👉 方案 A：让它用自带的浏览器硬读网页（免费，推荐）**
如果你只是想让它帮忙看某个网页，你可以直接在飞书里把网址甩给它：
> “帮我进入并且总结一下这个网页的核心观点：https://docs.openclaw.ai/”

*(注意：如果它还是报 browser 错误，请在你的电脑的管理员终端里，手动运行一次以下安装命令，来帮它把底层浏览器环境装好：)*
```bash
npx playwright install chromium
```

**👉 方案 B：为它装上真正的全球搜索引擎（化身真·万事通）**
如果你真的想让它像 Bing/ChatGPT 一样天天看新闻找资料，你需要去海外搜索引擎 [Brave Search API 官网](https://brave.com/search/api/) 申请一个免费的 Key（每月给 2000 次免费搜索，足够个人玩了）。
拿到 Key 之后，在你的电脑终端运行：
```bash
openclaw configure --section web
```
回车后，把那个免费的 Brave API Key 填进去，它就能随时联网搜索全宇宙的新闻了！

## 🚀 第二问：那我该怎么“玩”它？（核心场景体验）

这个机器人最变态的能力 **从来不是纯聊天（闲聊）**，而是 **“被授予了你电脑的最高级读写权限，并在本地帮你干程序员的活”**！

你现在就可以在飞书里，把复制下面这些魔法指令发给它，体验一下真正的 Agent（智能体代理）有多可怕：

### 场景 1：让它化身你的本地代码审计员
因为它刚才被开启了 `code-search` 技能，它能直接穿透飞书网络，看到你电脑此时此刻文件夹里的代码！
尝试在飞书里发送：
> “现在，就在你的终端当前目录 `D:\code\vb\claw` 下帮我搜一下，这个项目里主要有哪几个 JS 或 TS 配置文件？帮我分析它们的作用。”
> （它会在你电脑本地执行 grep/find 命令，阅读原代码，并把结果直接发到飞书里给你解答！）

### 场景 2：连 IDE 都不用打开，直接让它在电脑上写代码
不用打开代码编辑器，直接使唤它当黑客（它是有权限在本地跑脚本的）：
> “请在我的电脑桌面上（假设是 `C:\Users\levimin\Desktop`）新建一个叫 `test_api.py` 的文件，里面写一段使用 python 的 requests 库模拟请求百度首页的代码，并直接用命令行帮我运行测试一下，把结果截图反馈给我。”
> （它会自己思考语法，帮你保存到物理硬盘，甚至用你的电脑帮你运行！然后把输出信息发回给你。）

### 场景 3：电脑报错了直接扔给它排雷
当你在折腾啥服务报错了，不需要去浏览器里到处复制粘贴：
> “我的网关挂了，帮我直接读取一下当前目录下最新的也就是今天的 `openclaw.log` 运行日志文件，看看里面有没有什么 Error 关键字？用中文给我总结出错误原因和两套解决方案。”

### 场景 4：帮我看长篇离线文档
> “（丢一个本地绝对路径给它，比如：`D:\文档\年终报告.pdf`），这是一个几十页的长文件，你帮我读取一下，把核心结论提取成 3 个不带废话的加粗要点总结。”

### 🤯 场景 5：终极套娃（Agent Inception）
既然它有权限跑本地命令，而你本地又装了自带搜索能力的 `codex cli` 命令行——你可以让魔法打败魔法！
在飞书中对它说：
> “我的电脑本地安装了 `codex` 命令行工具。请你在我的终端里运行 `codex '搜索最近关于 openclaw 的新闻'` 命令，等待命令执行完毕后，把那边的搜索结果读取出来并总结发给我！”
> （它会在后台悄悄唤起另一个 AI，利用另一个 AI 的搜索能力拿到结果后再反馈给你！这就是所谓的“多智能体协作”套娃。）

这才是它真正的【智能体 Agent】正确打开方式！现在你也可以按照它在那条回复里的【方式 B】，甩两个关于 OpenClaw 的新闻链接给它，让它帮你看看，找找实操的感觉！🦞
