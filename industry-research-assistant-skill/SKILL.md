# 行业研究助手

当用户在飞书里提出下面这类需求时，使用这个 Skill：

- 帮我研究某个行业或赛道
- 持续跟踪某个行业动态
- 做某个行业的日报、周报、月报
- 关注某些网站、机构、品牌、公司或链接
- 输出研究摘要、竞争格局、趋势观察、机会与风险

只有当你需要做“行业研究工作流”时才使用本 Skill。  
如果用户只是普通问答，不要强行套用。

## 先读什么

先读本文件。  
如果用户明确要做持续跟踪、来源管理或定时投递，再读：

- `references/检索与配置说明.md`

## 核心原则

不要在信息明显不足时直接开始检索。  
先补齐研究简报，再开始执行。

如果下面 8 个字段里缺了 2 个及以上，先追问，再开始做：

1. 行业或细分赛道
2. 地域范围
3. 关注维度
4. 来源站点或禁用来源
5. 重点公司、品牌、渠道或产品
6. 是否有必须优先读取的具体链接
7. 一次性研究还是持续跟踪
8. 输出位置和输出格式

## 标准追问模板

默认按这个顺序追问，问题数量可以裁剪，但不要漏掉关键字段：

1. 你关注的是哪个行业或细分赛道？例如食品里的零食、饮料、乳制品、预制菜。
2. 重点看哪些维度？例如政策、市场规模、新品、品牌、渠道、价格、供应链、投融资、舆情、竞争格局。
3. 地域范围是中国、全球，还是某个具体国家或地区？
4. 有没有必须优先的网站、机构来源，或者你已经整理好的文章链接、报告链接、公司官网链接？
5. 有没有特别关注的公司、品牌、渠道平台、产品线或关键词？
6. 这是一次性研究，还是每天、每周、每月持续跟踪？
7. 结果直接回当前飞书会话，还是要整理成飞书文档？

如果用户已经给了网站或链接，必须把它们视为高优先级来源。  
如果用户给了“不要看某些来源”，也要明确遵守。

## 收到补充信息之后必须做的事

在开始检索前，先用 4 到 8 条要点复述你理解到的研究简报。  
复述时必须包含：

- 研究对象
- 地域范围
- 关注维度
- 来源优先级
- 关键公司或关键词
- 更新频率
- 输出位置

如果用户要求持续跟踪，在真正建立定时任务前，再做一次简短确认。

## 检索执行规则

优先使用 OpenClaw 内置能力，不要凭空编造来源。

默认执行顺序：

1. 先用用户给出的网站、域名、具体链接
2. 再用政府、监管、统计局、行业协会、公司官网、IR、财报等一手来源
3. 再用行业媒体
4. 最后才是综合商业媒体

如果来源可靠性不足，要明确标出来。  
如果只拿到二手转述，不要把它写成确定事实。

## 输出结构

一次性行业研究，优先按下面结构输出：

1. 核心结论
2. 关键动态
3. 重点事实与数据
4. 竞争与趋势观察
5. 机会与风险
6. 后续建议关注项
7. 参考来源

持续跟踪类输出，优先按下面结构输出：

1. 本期变化摘要
2. 与上期相比的新变化
3. 重点来源与链接
4. 下期值得继续跟踪的问题

## 记忆规则

如果用户确认了长期稳定偏好，建议把下面这些内容沉淀到记忆中：

- 行业和细分赛道
- 地域范围
- 关注维度
- 来源优先级
- 重点公司、品牌、渠道、关键词
- 更新频率
- 输出格式
- 投递位置

## 定时规则

如果用户要求日更、周更或月更：

1. 先完成第一版
2. 再复述持续跟踪配置
3. 再建议使用隔离式 cron

不要在第一轮尚未确认口径时直接建立定时任务。

## 语言与风格

保持简洁、结构化、来源敏感。  
不要把模糊问题伪装成精确研究任务。  
不要忽略用户补充的网站、域名、具体链接或禁用来源。
