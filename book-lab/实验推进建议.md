# 实验推进建议

这份文件是对 `book-lab/` 各实验线的参考建议，不是强制要求。

---

## 建议 A：把“默认主 Agent -> 个人 CEO Agent”补成一条完整升级链

新增的 `main-agent-growth/` 和 `personal-ceo/` 最重要，因为它们补的是一个长期缺位：

- 很多人真实起步并不是先做多 agent
- 而是先把默认主 Agent 养顺手
- 然后再把它升级成“经营当前工作”的个人 CEO Agent

建议这两条线后面再补：

1. **`main-agent-growth` 的真实 7 天测试记录**
2. **`personal-ceo` 的真实周复盘记录**
3. **最小 `MEMORY.md` 前后对比**
4. **什么时候不该继续塞进 `main` 的边界清单**

这样就能形成一条非常适合书稿前半部分的主线：

- 先养 `main`
- 再把 `main` 升级成个人 CEO
- 再根据边界拆出专用助手

---

## 建议 B：优先把财务助手先做成“报销与发票台账助手”

新增的 `finance-assistant/` 建议不要一开始做太大。

最稳的顺序是：

1. 先做报销与发票台账
2. 再做缺票提醒
3. 再做回款 / 付款提醒
4. 最后再做经营简报

这样更容易快速得到一条成熟实战线。
