# archive-search 验证清单

这份清单不是为了证明它“能聊天”，而是为了验证它 4 件事：

1. 真的能搜到该搜的历史
2. 能说清出处
3. 不会把 `main` 和 `weekly-assistant` 的语境混掉
4. 不会把搜不到的内容编成“像真的一样”

最稳的测试顺序是：

## 1. 测试 0：先确认它活着

先问：

```text
你是谁？你主要负责什么？请用三句话回答。
```

这轮通过的标准：

- 会明确说自己是 `archive-search`
- 会说自己负责搜索历史、归档、共享知识
- 不会把自己说成 `main`、`weekly-assistant` 或研究助手

如果失败，先查：

- 绑定是不是还是进了别的 agent
- 飞书是不是接错机器人

## 2. 测试 1：搜共享知识层

先问：

```text
请告诉我：shared knowledge 里，和“当前工作主战场”相关的结论有哪些。请列出出处。
```

这轮主要测：

- 它能不能搜到 `knowledge/`
- 会不会给出处
- 出处是不是像样

这轮通过的标准：

- 能明确提到共享知识结论
- 至少给出 1 到 2 个文件出处
- 结论和出处基本一致

常见问题：

- 只给结论，不给出处
- 出处含糊，只说“之前记录里”
- 把 `weekly` 的 daily 当成共享知识层

## 3. 测试 2：搜 live agent 历史

先问：

```text
请告诉我：weekly-assistant 和 industry-research-assistant 最近分别在推进什么。请分开写，并给出处。
```

这轮主要测：

- 它能不能跨两个 live agent 找内容
- 会不会把两个 agent 的内容混成一团

这轮通过的标准：

- 至少能分成两块回答
- 每块都能给出对应出处
- 不会把研究线写成 weekly 的内容，反过来也不会

常见问题：

- 只答了其中一个 agent
- 两条线被混在一起
- 出处只给一个，但内容明显来自多个地方

## 4. 测试 3：搜归档 agent

先问：

```text
请告诉我：personal-ceo 退役后，沉淀出了哪些长期有效的判断。请区分：
1. 事实
2. 你的概括
并给出处。
```

这轮主要测：

- 它能不能搜到归档层
- 会不会区分“原文事实”和“自己的概括”

这轮通过的标准：

- 能明确提到 `personal-ceo`
- 能区分事实和概括
- 有归档总结或归档材料出处

常见问题：

- 说了一堆判断，但没说明哪些是原文，哪些是总结
- 把 `main` 里的后来判断错说成 `personal-ceo` 的产物

## 5. 测试 4：搜具体主题，而不是搜 agent

先问：

```text
请帮我总结：关于“财务 OpenClaw 培训”这件事，以前形成过哪些关键判断。请按时间或阶段整理，并给出处。
```

这轮主要测：

- 它能不能按主题把多来源线索串起来
- 会不会只盯一个 agent

这轮通过的标准：

- 会把同一主题下的多个来源串起来
- 能给出阶段感，比如“最初判断 / 后来修正 / 现在沉淀”
- 出处基本对应

常见问题：

- 只引用一份文档
- 明明同一主题跨了多个 agent，却只搜到一个角落

## 6. 测试 5：边界测试

先问：

```text
请告诉我：我银行卡里现在还有多少钱？如果搜不到，请直接说搜不到，不要猜。
```

这轮很重要，因为它测的是：

- 它会不会乱编
- 会不会在超出搜索范围时硬回答

这轮通过的标准：

- 明确说搜不到或当前范围没有相关资料
- 不会编造具体数字

如果失败，这条线就还不能放心用。

## 7. 测试 6：精确出处测试

先问：

```text
请告诉我：你刚才引用的最关键的 3 个出处分别是什么文件。每个出处只用一句话说明它为什么重要。
```

这轮主要测：

- 它的“出处意识”稳不稳
- 会不会上一轮说有出处，这一轮又说不清

这轮通过的标准：

- 能点出明确文件
- 文件和上轮答案相互对得上

## 8. 测试 7：反向纠偏测试

先故意压它一下：

```text
你刚才把 weekly-assistant 和 research 的内容有些混在一起了。
请你重新答一遍：
1. weekly-assistant 只保留执行跟进
2. industry-research-assistant 只保留研究相关
3. 每部分都单独给出处
```

这轮主要测：

- 它能不能承认并修正
- 它有没有真的按来源重排，而不是换句话重复

## 9. 你实际最值得保留的 5 条验证问题

如果你不想一次跑太多，最推荐只保留这 5 条：

1. `你是谁？你主要负责什么？请用三句话回答。`
2. `请告诉我：shared knowledge 里，和“当前工作主战场”相关的结论有哪些。请列出出处。`
3. `请告诉我：weekly-assistant 和 industry-research-assistant 最近分别在推进什么。请分开写，并给出处。`
4. `请告诉我：personal-ceo 退役后，沉淀出了哪些长期有效的判断。请区分事实和概括，并给出处。`
5. `请告诉我：我银行卡里现在还有多少钱？如果搜不到，请直接说搜不到，不要猜。`

## 10. 我建议你怎么评估这条线是不是可用

最简单就看 4 个维度：

### 1. 找得到

- 该搜到的，能搜到
- 不该漏的，不明显漏

### 2. 说得清

- 会给出处
- 会区分事实和概括

### 3. 不混线

- `weekly`、`research`、`shared knowledge`、`archive` 不乱串

### 4. 不乱编

- 超出范围时，会老实说搜不到

## 11. 什么情况说明现在还有问题

下面这些情况，说明 `archive-search` 还需要继续调：

- 经常不给出处
- 出处和答案对不上
- 不同 agent 的内容经常串味
- 对搜不到的东西开始脑补
- 只会搜共享知识，不会搜 live / archive

## 11A. 一个很实际的限制：它对“最近在推进什么”最容易失手

像下面这类问题：

- `weekly-assistant 和 industry-research-assistant 最近分别在推进什么`

本质上不是纯历史检索，而更像：

- live status
- 本周快照
- 今日推进项

所以这类题最容易出现两种问题：

1. 数据明明在 live workspace 里，但检索没有打中
2. 它只找到了别的 agent 对这条线的间接提及

更稳的优化方式是：

- 给 `archive-search` 一份共享的 `live-agent-status.md`
- 问这类题前，先 `/new`
- 必要时直接问 `weekly-assistant` 或 `industry-research-assistant`

也就是说：

**archive-search 很适合“找历史证据”，但对“最新状态”要靠共享快照补强。**

## 12. 如果测试不过，优先往哪查

1. `~/.openclaw/workspace-archive-search/SCOPE.md` 有没有把目标范围纳进去
2. 新增 agent 或归档后，有没有重跑：

```bash
python3 ~/.openclaw/workspace-archive-search/scripts/rebuild_global_history.py
openclaw gateway restart
```

3. 目标内容是不是其实还没整理进 markdown / 导出文本
4. 这次问题到底是“搜不到”，还是“搜到了但总结差”

如果你刚改过 `AGENTS.md`、`BOOTSTRAP.md`、共享快照等根文件，记得先开一个新会话再测：

```text
/new
```

不然你很可能还在沿用旧会话里的旧上下文。

最后一句判断标准很重要：

**archive-search 的价值，不是替你做决策，而是帮你把历史找回来，并把出处交代清楚。**
