# 默认主 Agent 养成：7 天验证手册

这份手册不是概念说明，而是一份可以直接照着跑的验证手册。

它的目标是：

`默认主 Agent -> 连续日常互动 -> 纠偏收敛 -> 补最小显式记忆 -> 判断是否该拆分`

## 1. 这轮实验最终要帮你得到什么

做完这 7 天，你至少应该得到下面 4 个结果：

1. 哪些高频任务用 `main` 已经足够顺手
2. 哪些偏好只靠聊天能慢慢养出来
3. 哪些内容必须显式写进 `MEMORY.md`
4. 什么时候应该从“继续养 `main`”切到“拆专用 agent”

## 2. 开始前先记住这轮实验的边界

这轮实验默认：

- 先用默认主 Agent
- 不先新建专用 agent
- 不先写 Skill
- 不先做 cron
- 不先大改 `AGENTS.md`

如果你已经做过少量基础配置，不用回退；  
但这轮实验的重点仍然是：**先靠日常对话验证 `main` 的自然养成能力。**

## 3. 这 7 天建议固定记录什么

建议在默认工作区额外留一份简单记录，例如：

```text
~/.openclaw/workspace/main-agent-growth-log.md
```

每天固定记 4 件事：

1. 今天拿它做了什么任务
2. 哪条输出已经明显更贴近你的习惯
3. 哪条输出还是偏了
4. 你觉得这个任务还适不适合继续留在 `main`

## 4. 每天都尽量用同一套观察维度

每天都优先看下面 5 个维度：

1. 输出风格有没有更稳定
2. 它会不会重复犯同一类错误
3. 纠偏后下一轮有没有收敛
4. 跨轮对话有没有延续感
5. 任务边界有没有开始变乱

## 5. 第 1 天：先验证“好不好沟通”

第一天只做高频轻任务，例如：

- 整理一段零散笔记
- 起草一段飞书回复
- 总结今天做了什么

这一天重点不是看它多聪明，而是看：

- 你说一次风格要求后，它能不能跟上
- 你改一次后，它会不会立刻收

建议这样测：

1. 先让它整理一段零散记录
2. 明确告诉它你要的风格
3. 再让它整理第二段相似材料

理想结果：

- 第二轮已经比第一轮更像你要的样子

## 6. 第 2 天：验证“重复任务会不会越来越顺”

第二天继续拿相似任务来测，不要一下换很多类型。

最适合的是：

- 再做一次摘要
- 再做一次回复草稿
- 再做一次行动项整理

重点看：

- 它有没有开始抓住你要的结构
- 你是不是已经不用每次都把风格讲一遍

## 7. 第 3 天：验证“跨天连续性”

第三天重点不是任务本身，而是看跨天后：

- 它还能不能接住你常见的工作方式
- 你昨天反复强调的偏好，今天是不是还在

这一天可以故意做两件事：

1. 先接着昨天的任务风格继续做一轮
2. 再发一个新任务，看它会不会沿用你已经磨出来的表达习惯

## 8. 第 4 天：故意做一次明显纠偏

第四天建议故意制造一个你不满意的输出，然后明确纠偏。

例如：

- 让它别写太长
- 别给模板
- 先结论后展开
- 别上来讲大道理

重点看：

- 同一轮纠偏能不能立刻生效
- 后面一轮是否还保留这次纠偏结果

如果同类错误反复出现，就记下来。  
这往往说明：这个偏好只靠对话养还不够稳。

## 9. 第 5 天：补最小 `MEMORY.md`

到第五天，再补一版最小显式记忆。

只建议补这类内容：

- 默认用中文
- 偏好简洁、结构化输出
- 常见工作类型
- 一两条长期稳定工作方式

不要补：

- 今天做了什么
- 本周待办
- 零散短期任务
- 临时项目细节

这一天重点看的是：

- 补了最小记忆后，输出是不是明显更稳
- 哪些以前靠聊天容易飘的点，现在是不是固定下来了

## 10. 第 6 天：开始混一点异质任务

第六天可以开始轻微混任务，例如：

- 上午做整理
- 下午做回复草稿
- 晚上做今天总结

这一步不是为了故意把它搞乱，而是为了看：

- `main` 在任务种类开始增多时，是否仍然清晰
- 哪些任务已经开始互相污染表达和上下文

## 11. 第 7 天：做最终判断

第七天不再继续堆任务，而是回看前 6 天记录。

重点回答 3 个问题：

1. 哪些任务继续留在 `main` 最划算
2. 哪些任务虽然能做，但已经开始拖脏 `main`
3. 哪些任务从现在开始更适合拆成专用 agent

## 12. 什么时候说明这条线跑通了

如果下面这些事都发生了，说明这条线已经成立：

- 你已经连续几天自然地把轻任务交给 `main`
- 你能明确看到“连续对话 + 纠偏”带来的收敛
- 你能分清纯对话养成和显式记忆沉淀的边界
- 你能说出至少 3 类适合继续留在 `main` 的任务
- 你能说出至少 2 类该拆成专用 agent 的任务

## 13. 什么时候说明该停下来拆分了

如果出现下面这些情况，就不要再强行把一切都留在 `main`：

1. 任务类型已经明显分化
2. 你开始需要独立工作区
3. 你开始需要独立 cron 或提醒
4. 你开始需要更强的规则隔离
5. 你发现一个任务长期在污染另一个任务的上下文

## 14. 一句话总结

这条实验线真正想验证的不是：

**默认主 Agent 能不能包打天下。**

而是：

**默认主 Agent 完全可以成为起步阶段最自然的使用入口；只有当任务边界和长期结构开始出现时，拆专用 agent 才真正划算。**
