# 个人 CEO Agent：结合当前工作的验证手册

这份手册不是概念说明，而是一份第一轮可以直接照着跑的验证手册。

它的目标不是：

- 让 Agent 给你做宏大人生规划

而是：

- 让 Agent 先围绕你当前工作做经营判断

也就是先把下面这条链路跑通：

`当前工作盘子 -> 当前项目与收入线 -> 本周资源分配 -> 本周经营判断 -> 下周取舍`

## 1. 第一轮验证范围

第一轮先验证：

- 识别你当前的工作主线
- 识别当前项目 / 客户 / 交付
- 识别当前收入与机会通道
- 给出本周资源分配建议
- 给出该停、该延、该推进的判断
- 形成一版周复盘

第一轮先不强求：

- 远期人生规划
- 很复杂的自动化
- 独立 agent
- 复杂 cron

## 2. 开始前先准备什么

第一轮建议你至少准备下面两类输入：

1. 你当前到底在做哪些工作
2. 你当前主要靠哪些项目、客户、产品或能力产生收入和机会

如果你现在还没有整理过，可以先让它帮你做第一版。

## 3. 第一轮推荐先固定哪几类信息

第一轮建议至少先固定下面这些信息：

1. 你当前主要工作类型
2. 当前在推进的项目 / 客户 / 交付
3. 当前最重要的收入通道
4. 当前最耗时间的事情
5. 当前最容易拖住你的瓶颈
6. 这周最想拿到的结果

这些信息比空泛规划重要得多。

## 4. 第一轮建议怎么开场

第一轮可以直接对默认主 Agent 发这样一段：

```text
这次我不想把你只当普通助手，而是想让你先作为“经营我当前工作的个人 CEO Agent”来协助我。

先不要给我很大的长期规划，也不要上来讲人生目标。

你先只围绕我当前的工作来判断：
1. 我现在真正的主战场是什么
2. 哪些项目、客户、交付和收入线最值得优先投入
3. 哪些事情在占用我的时间，但未必值得继续投
4. 这周我最该推进的 1 到 3 件事是什么

如果信息不够，请先问我最多 5 个关键问题，把“当前工作盘子”先问清楚。
```

## 5. 第一轮通过标准

如果下面这些事情都发生了，说明这条线方向是对的：

- 它没有一上来跑到远期人生规划
- 它先围绕你当前工作的现实情况发问
- 它关心项目、客户、收入、交付和资源投入
- 它已经开始做“取舍判断”，而不是只帮你整理

## 6. 第二轮验证：给它真实工作材料

当第一轮问答方向对了，再给它真实工作材料。

最适合的材料包括：

- 当前项目列表
- 当前客户与合作线
- 当前交付清单
- 当前收入来源
- 当前卡住的问题

你可以直接发这种结构：

```text
我当前的实际情况大概是这样：

1. 当前主要工作：
2. 当前在推进的项目 / 客户：
3. 当前最接近收入的事情：
4. 当前最耗时间的事情：
5. 当前最卡的瓶颈：
6. 这周最想拿到的结果：

请你先不要替我列很多执行细节。
先帮我判断：
1. 现在最该押注什么
2. 什么应该先暂停、延后或降投入
3. 这周最值钱的 1 到 3 个动作是什么
```

## 7. 第三轮验证：让它写一版“当前工作经营判断”

如果前两轮已经像样，再让它把判断写成一个工作文件。

建议至少写到：

```text
ops/current-work.md
ops/active-projects.md
```

你可以这样说：

```text
请把我们刚才确认过的内容写成一版当前工作经营判断，至少写到：
1. ops/current-work.md
2. ops/active-projects.md

要求是：
1. 先写清我当前工作的主战场
2. 再写清当前在推进的项目、客户、交付
3. 再写清当前最值得押注的方向和需要收缩的方向

写完后告诉我：
1. 分别写了哪些文件
2. 每个文件的核心内容是什么
```

这一步重点看：

- 它有没有真正落盘
- 写出来的内容是不是经营判断，而不是流水账

## 8. 第四轮验证：让它做一版本周经营决策

当前工作判断写稳之后，再让它给一版本周经营决策。

例如：

```text
请基于我们已经确认过的当前工作盘子，给我一版本周经营决策。

重点回答：
1. 本周最该押注的 3 件事
2. 本周最该减少投入的 2 件事
3. 哪个项目最接近结果或收入
4. 哪个动作如果不做，下周会更被动
5. 我这周最该守住的边界是什么
```

这一步的目标不是看它会不会写激励话术，而是看：

- 它会不会做真正的资源分配判断

## 9. 第五轮验证：做一版周复盘

一周之后，再拿真实结果回头测它。

建议这样问：

```text
这是我这一周真实推进的情况：

1. 做成了什么
2. 哪些没做成
3. 哪些事花了很多时间但不值
4. 哪些动作最接近结果或收入
5. 下周可能继续推进什么

请你不要只帮我总结，而是站在个人 CEO 的角度判断：
1. 这周最值得保留的投入是什么
2. 这周最该停止或收缩的投入是什么
3. 下周继续押注什么最划算
```

## 10. 如果第一轮表现不好，优先看什么

如果这条线跑偏，优先看这几层：

1. 它是不是跑去讲抽象长期规划
2. 它是不是只会给一般性建议
3. 它是不是没有围绕当前项目、客户、收入和交付来判断
4. 它是不是没有做真正的取舍
5. 你给它的当前工作信息是不是还太空

## 11. 什么时候说明这条线已经成立

如果下面这些事都发生了，说明这条线已经成立：

- 它已经不只是帮你执行，而是开始帮你判断
- 它会围绕当前工作做资源分配和优先级取舍
- 它能指出“该做”和“别做”的区别
- 它会把职业、收入、交付放在同一个经营框架下看
- 你能明显感到：它开始像一个经营代理，而不只是助手

## 12. 什么时候该继续留在 `main`

下面这些情况下，继续放在 `main` 是合理的：

- 你还在验证这套角色是否适合你
- 当前工作结构还不算太复杂
- 你想先把经营判断习惯磨出来
- 你还不需要独立的长期自动化

## 13. 什么时候应该拆出专用 personal-ceo / operator agent

如果后面出现下面这些情况，就适合拆出去：

1. 你已经形成稳定经营框架
2. 你每天都在用这条线做判断
3. 你需要单独工作区保存长期决策材料
4. 你希望把普通助理任务和经营判断彻底隔开
5. 你后面要给它加复盘、提醒、经营周期机制

## 14. 一句话总结

这条实验线真正想验证的不是：

**OpenClaw 会不会假装成你的老板。**

而是：

**默认主 Agent 能不能逐步升级成一个围绕你当前工作做取舍、分配和经营判断的个人 CEO Agent。**
