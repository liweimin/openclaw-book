# 个人 CEO Agent 实验蓝图

这份蓝图不是正式书稿章节，而是“把默认主 Agent 升级成个人 CEO”这条实验线的设计稿。

它要回答的是：

- 默认主 Agent 能不能不只是做助手，而是开始经营你当前的工作
- 这条线怎样避免滑向空泛的人生规划
- 个人 CEO 到底该围绕哪些现实变量做判断
- 什么边界下它仍然适合留在 `main`，什么边界下应该拆出去

## 1. 这条实验要做成什么

这条实验想做的不是：

- 一个会给你讲大道理的人设
- 一个天天催你努力的“电子老板”
- 一个替你拍所有人生板的人

而是：

**一个以你当前工作为经营对象的主 Agent。**

它的关注重点应该是：

1. 你现在正在做什么工作
2. 哪些项目、客户、交付和收入通道最重要
3. 你当前时间和精力投得值不值
4. 这周最该推进什么，最该砍掉什么
5. 哪些动作最有可能带来更好的职业位置和更高收入

## 2. 为什么这条线值得单独做

如果只是“普通助手”，它更像：

- 帮你整理
- 帮你起草
- 帮你总结

这些当然有价值，但还不够构成真正的经营角色。

而你要的这条线更像：

- 站在经营者角度看你当前工作
- 不只是接收任务，而是判断资源投向
- 不只是完成待办，而是持续优化你的职业与收入结构

这件事比“普通助手”更难，但也更值得写进书。

## 3. 这条线的第一性问题是什么

个人 CEO Agent 的第一性问题不是：

- 这件事怎么做

而是：

- 这件事值不值得做
- 现在是不是最该做
- 做完会不会更接近你当前想要的职业和收入结果

所以它的工作顺序应该是：

1. 先看目标
2. 再看资源
3. 再看机会成本
4. 最后才是执行动作

## 4. 这条线默认先围绕什么做

第一版不要上来就谈很大的 3 年规划。

第一版默认只围绕这几类现实东西：

1. 当前主要工作
2. 当前在做的项目或客户
3. 当前手里的交付压力
4. 当前收入通道
5. 当前最缺的资源
6. 这周和这个月的现实目标

换句话说：

**先经营当前盘子，不先空谈远方。**

## 5. 这条线默认用什么承载

第一版建议：

- 继续用默认主 Agent
- 不急着新建专用 agent

原因很简单：

- 这条线本来就是在验证 `main` 能不能升级成经营代理
- 如果一开始就拆出去，就失去这条实验的意义了

## 6. 这条线最该验证哪些判断能力

第一版最该验证 6 件事：

1. 它能不能说清你当前工作的主战场是什么
2. 它能不能识别哪些事情只是在占用精力，但不创造结果
3. 它能不能给出“本周最该推进的 1 到 3 件事”
4. 它能不能识别收入、成长、交付三者之间的冲突
5. 它能不能把“想法”转成“经营动作”
6. 它能不能在复盘时指出你这周真正赚到的，不只是钱，也包括机会和位置

## 7. 推荐的最小工作区结构

这条线虽然先用 `main`，但仍然建议给它最小经营材料。

第一版建议逐步沉淀这些文件：

```text
ops/
├─ current-work.md
├─ active-projects.md
├─ revenue-pipeline.md
├─ weekly-review.md
└─ decision-log.md
```

最简单的理解是：

- `current-work.md`
  - 你当前到底在干什么
- `active-projects.md`
  - 手上在推进的项目、客户、交付
- `revenue-pipeline.md`
  - 哪些事情正在或可能产生收入
- `weekly-review.md`
  - 这一周到底做得值不值
- `decision-log.md`
  - 重要取舍和判断，避免反复摇摆

第一轮不需要一次全配满。  
先有 `current-work.md` 和 `active-projects.md` 就够了。

## 8. 这条线的日常节奏应该长什么样

### 每天

重点不是“列很多待办”，而是：

- 今天最该打哪一仗
- 哪件事最接近结果
- 哪件事最该停

### 每周

重点不是“回忆都做了什么”，而是：

- 哪些动作真正带来推进
- 哪些投入是低回报
- 下周最值得继续押注什么

### 每月

重点不是空泛复盘，而是：

- 这个月收入和机会有没有变好
- 哪条工作线值得继续加码
- 哪条线应该收缩或停止

## 9. 这条线最容易跑偏到哪里

第一版最容易跑偏到 3 个方向：

### 跑偏 1：变成说教型老板

如果它只会：

- 要你自律
- 要你努力
- 要你坚持

那这条线就废了。

### 跑偏 2：变成空泛规划师

如果它总在说：

- 三年目标
- 人生愿景
- 战略宏图

却对你这周的真实工作没有判断，那也没用。

### 跑偏 3：变回普通助手

如果它只是：

- 帮你总结
- 帮你列待办
- 帮你润色文案

却不判断取舍，它仍然只是助手，不是 CEO Agent。

## 10. 第一轮成功标准

第一轮实验通过，至少要满足：

- 它能围绕你当前工作，而不是上来就谈大规划
- 它能识别 1 到 3 个当前最重要的经营焦点
- 它能指出哪些事情占用资源但不值得继续投入
- 它能在周复盘里给出有判断力的取舍，而不是流水账
- 你能明显感到：它已经不只是“帮你干活”，而是在“帮你经营”

## 11. 什么时候这条线该继续留在 `main`

如果下面这些条件成立，继续留在 `main` 是合理的：

- 你还在验证这套角色是否成立
- 当前工作线还不算特别复杂
- 你还在磨“经营判断”的风格
- 你还不需要独立提醒和复杂自动化

## 12. 什么时候该拆出专用 CEO / Operator agent

如果后面出现下面这些情况，就可以考虑拆出去：

1. 你已经形成固定经营框架
2. 你需要独立工作区沉淀长期决策
3. 你要接更强的复盘和定时机制
4. 你不想让普通助理任务污染经营判断上下文
5. 这条线开始成为你每天固定使用的“决策中枢”

## 13. 这条线后面怎么服务整本书

如果实验稳定，这条线很适合变成书里的一个关键主张：

> OpenClaw 的默认主 Agent 不只能做个人助手，它还可以在真实工作中逐步升级成一个“经营你当前工作”的个人 CEO Agent。

这会把整本书的定位从：

- 会用 OpenClaw

往前推进到：

- 会把 OpenClaw 养成一个真正参与决策的工作系统
