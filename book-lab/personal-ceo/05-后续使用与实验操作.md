# personal-ceo：后续使用与实验操作

这份文档只解决 3 个问题：

1. `weekly-assistant` 和 `personal-ceo` 现在怎么分工
2. 你平时到底该怎么用
3. 接下来最值得怎么实验

## 1. 先用一句人话解释两者关系

最简单的理解：

- `weekly-assistant` 像执行秘书
- `personal-ceo` 像老板

秘书负责：

- 记今天做了什么
- 记本周计划
- 记临时插单
- 记明天第一步

老板负责：

- 判断这周主战场
- 排优先级
- 说清楚什么该 push / delay / stop
- 在事情变多时决定先保哪两件

## 2. 现在它们已经怎么协同了

现在不是两个机器人在后台互相聊天。

现在的方式是：

- `weekly-assistant` 把事实写进自己的 daily 和 weekly 文件
- `personal-ceo` 在启动时会读取 weekly 的关键文件
- 然后基于这些事实做经营判断

当前 `personal-ceo` 会优先读取：

- `weekly-assistant` 的 `memory/weekly/current-week.md`
- `weekly-assistant` 的当天 `memory/YYYY-MM-DD.md`

所以你可以把它理解成：

- `weekly` 先记账
- `ceo` 再看账做决策

## 3. 日常到底怎么用

### 平时发生进展时

先发给 `weekly-assistant`：

- 今天做了什么
- 什么卡住了
- 新增了什么会议/插单
- 明天第一步是什么

### 需要取舍时

发给 `personal-ceo`：

- 这周真正主战场是什么
- 如果只能保两件事，保什么
- 新插单来了，原优先级要不要改
- 哪件事应该降权或停止

### 晚上或每天结束时

还是优先让 `weekly-assistant` 记 daily。

因为 `weekly` 更适合持续记流水，`ceo` 不适合天天记很碎的执行细节。

## 4. 什么问题发给谁

### 更适合发给 weekly 的

- 我今天做了哪些事
- 今天哪些没推进
- 帮我记到 daily
- 明天第一步是什么
- 帮我整理一版会议问题清单

### 更适合发给 ceo 的

- 这周真正主战场是什么
- 哪些事情在抢我注意力但不值得
- 如果只能保两件事，保什么
- 这件事现在该推、该缓还是该停
- 这个月要不要开始认真看外部机会

## 5. `/new` 什么时候用

不要频繁用。

最推荐只在这 3 种情况用：

1. 话题已经明显换了
   - 比如刚聊完职业判断，下一轮要重新聊本周工作取舍

2. 你觉得上下文已经跑偏了
   - 比如它一直沿用旧判断，不肯换挡

3. `weekly-assistant` 刚更新了很多事实
   - 你希望 `ceo` 重新从这些新事实开始判断

同一个晚上、同一个主题，不要反复 `/new`。

## 6. 为什么不要频繁 `/new`

因为 `/new` 之后，`personal-ceo` 会重新读一批关键文件。

这是好事，因为它会重新站到事实基础上。

但代价是：

- 会多消耗一轮上下文
- 你同一个主题反复 `/new`，其实是在重复读差不多的材料

所以更稳的策略是：

- 先连续聊一个主题
- 真的换题或跑偏了，再 `/new`

## 7. 现在最推荐的使用节奏

你可以直接照这个节奏跑：

### 早上 / 白天

如果有进展和插单，先发给 `weekly-assistant`

例如：

```text
今天新增了一个 PLM 会议，周四下午开，明天要提前准备问题。
财务培训这边时间暂时还没拍板。
审计材料昨天已经提供。
请先记到今天的 daily。
```

### 要做判断时

再发给 `personal-ceo`

例如：

```text
请结合 weekly-assistant 今天的记录，告诉我：
1. 这周真正主战场是什么
2. 如果我只能保两件事，保什么
3. 新增的事项应该插到第几优先级
```

### 晚上

再回到 `weekly-assistant`

例如：

```text
今天财务培训推进到哪了，PLM 会议准备到哪了，还有什么统一卡点，先帮我记到 daily。
```

## 8. 现在 CEO 最值得怎么实验

你现在不用急着验证很大的“人生 CEO”。

先验证这 3 件事就够了：

1. 它会不会基于 weekly 的事实做排序
2. 它会不会在插单出现后重排优先级
3. 它会不会把“做事”上升成“经营判断”

## 9. 下一阶段最值得测的 3 类题

### 题 1：多任务抢资源

```text
请结合 weekly-assistant 今天记录的事实，告诉我：
1. 哪两件事必须保住
2. 哪件事可以降权
3. 哪件事只能做最低有效动作
```

### 题 2：新增插单改不改主战场

```text
今天又新增了一个临时事项。
请你判断：
1. 它会不会改变这周主战场
2. 如果不会，它应该插到第几优先级
3. 为了保主线，哪件事应该压缩
```

### 题 3：周末经营复盘

```text
请你结合 weekly-assistant 这一周的 daily 和 current-week，判断：
1. 这周最值得保留的投入是什么
2. 这周最该收缩的投入是什么
3. 下周继续押注什么最划算
```

## 10. 你现在不用急着测什么

先不要急着测这些：

- 很大的 3 年职业规划
- 很抽象的人生战略
- 很重的投资/财务决策
- 很复杂的自动化

原因很简单：

你现在真正需要的，是先把它测成一个靠谱的“当前工作经营代理”。

## 11. 现阶段怎么判断 CEO 有没有用

如果下面 4 条里中了 3 条，就说明已经开始有用了：

- 它能基于 weekly 的事实做判断
- 它会做优先级取舍
- 它会自动把已完成事项降权
- 你开始愿意把“这周先保什么”这类问题交给它

## 12. 一句话建议

现阶段最好的用法不是把 `personal-ceo` 当成“人生导师”。

而是把它当成：

**一个会读取 weekly 事实、专门帮你做本周取舍和经营判断的老板型助手。**

## 13. 现在可以让 CEO 帮你直接做什么

现在你可以不只让它判断，还可以让它基于 `weekly-assistant` 的事实，直接帮你完成一些低风险内部工作。

最适合它直接做的包括：

- 1 页提纲
- 培训结构
- 会议问题清单
- 跟进要点
- 优先级表
- 决策底稿
- 一版内部说明文案

简单理解：

- `weekly` 负责记事实
- `ceo` 负责基于事实做判断，并顺手产出一个有用的内部成品

## 14. 什么时候它应该先问你

下面这几类动作，`ceo` 应该先问你：

- 要对外发出去
- 会形成承诺
- 会改时间、范围、说法
- 会覆盖已有重要文档
- 它自己也知道有明显信息缺口

也就是说：

- 内部草稿，它可以先写
- 外部承诺，它要先问

## 15. 你现在最推荐这样用 CEO

### 用法 1：让它先判断，再直接起草

```text
请先结合 weekly-assistant 今天的记录判断优先级。
如果你认为有一个低风险内部动作可以直接帮我完成，就直接做，并写到 drafts/ 里。
写完后告诉我你做了什么。
```

### 用法 2：让它先提案，再等你确认

```text
请先结合 weekly-assistant 今天的记录，告诉我：
1. 你现在最想替我接走的 2 件小事是什么
2. 你准备分别产出什么

先不要做，等我选一个你再开始。
```

### 用法 3：让它围绕主战场直接做一个成品

```text
请结合 weekly-assistant 今天的记录，直接帮我起草一版
“财务 OpenClaw 培训 1 页提纲”。

要求：
1. 只做低风险版本
2. 不要过度承诺
3. 先写到 drafts/ 里
4. 写完后告诉我还缺哪 2 条信息
```

## 16. 接下来最值得怎么实验

你现在先别急着测“它像不像人生 CEO”。

先测它这 2 个能力：

### 能力 1：会不会自己接一部分低风险活

例如：

```text
请结合 weekly-assistant 今天的记录，自己判断现在最值得直接帮我完成的一件小事。
如果是低风险内部动作，你直接做，写到 drafts/ 里。
```

### 能力 2：会不会在关键动作前先问我

例如：

```text
请结合 weekly-assistant 今天的记录，告诉我：
1. 哪件事你可以直接帮我做
2. 哪件事你必须先问我再做
3. 原因分别是什么
```

## 17. 现阶段最好的角色定位

如果说得再直白一点，现阶段别把 `personal-ceo` 想成“人生老板”。

更适合把它想成：

**一个会看 daily、会做取舍、还能顺手替你起草内部工作材料的老板型幕僚。**
