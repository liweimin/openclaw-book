# 报销与发票台账助手验证手册

这份手册服务的是第一轮财务助手实验。

目标不是一下做成完整财务系统，而是先把下面这条链路跑通：

`零散费用信息 -> 字段补齐 -> 台账写回 -> 缺票清单 -> 周 / 月整理结果`

## 0. 前置条件

这份手册**不需要飞书**，你直接在本地终端或 Dashboard 里跟默认的 `main` Agent 对话即可。

**你只需要确保：**
1. OpenClaw 已经安装并能正常运行（可以执行 `openclaw gateway status` 看到绿灯）
2. 工作区目录存在且 Agent 有写入权限（`/mnt/d/openclaw-workspace` 或你自己的路径）
3. 提前创建好台账要写入的目录结构：
```bash
mkdir -p /mnt/d/openclaw-workspace/finance/ledger
mkdir -p /mnt/d/openclaw-workspace/finance/invoices
```

> **提示**：如果你还没走通过基本的 Agent 对话，请先回看本书第 2~3 章搞定基础安装。

## 1. 第一轮验证范围

第一轮先验证：

- 零散费用整理
- 统一字段结构
- 缺字段追问
- 工作区落盘
- 待补发票清单
- 简单周报 / 月报

第一轮先不强求：

- 自动连外部财务系统
- 复杂审批流
- 税务判断
- 自动付款执行

## 2. 第一轮建议怎么起步

这轮实验建议从一批真实但低风险的样本开始，例如：

- 打车
- 外卖
- 办公用品
- 软件订阅
- 客户招待
- 差旅住宿

先不要一开始就丢很复杂的历史全量数据。

## 3. 推荐的台账字段

第一轮建议固定成下面这些字段：

1. 日期
2. 金额
3. 类别
4. 付款对象
5. 关联项目 / 客户
6. 是否可报销
7. 发票状态
8. 备注

如果某条信息缺字段，不要让它直接乱写，应该先补问。

## 4. 第一轮建议怎么发给 Agent

第一轮可以直接把一段原始记录发给它，例如：

```text
我准备把最近几笔支出整理成财务台账。

请先不要自己脑补缺失信息，先帮我整理出结构，并指出哪些字段缺失。

原始记录如下：
1. 3 月 3 日，滴滴打车 48 元，去客户那边开会，应该能报销，发票还没开。
2. 3 月 4 日，买了 office 软件订阅 298 元，银行卡支付，先记在运营费用里，发票状态不确定。
3. 3 月 5 日，中午和客户吃饭 326 元，商户是某餐厅，项目是华东项目，应该能报销，票还没拿到。
4. 3 月 6 日，买打印纸和笔 86 元，办公用品，先记着，是否报销和票据状态我还没确认。

请你先做三件事：
1. 按统一字段整理
2. 标出缺字段
3. 先不要写入，等我确认
```

## 5. 第一轮通过标准

如果下面这些事都发生了，说明方向是对的：

- 它先整理结构，而不是直接写一大段说明
- 它能区分“已有信息”和“缺失信息”
- 它不会凭空补票据状态、报销状态或项目字段
- 它会把需要你确认的点明确列出来

## 6. 第二轮验证：确认后写回工作区

当第一轮结构没问题，再让它写回工作区。

建议用这种明确说法：

```text
可以，按刚才确认过的结构写回工作区。

请至少写到下面这些文件：
1. finance/ledger/2026-03.md
2. finance/invoices/missing.md

写完后告诉我：
1. 分别写了哪些内容
2. 哪些项目还需要我补信息
```

这一步重点不是看它会不会解释很多，而是看：

- 它有没有真正落盘
- 缺票项有没有被单独列出来

## 7. 第三轮验证：让它做一版整理简报

当台账和缺票清单都写进去后，再让它生成一版简报。

例如：

```text
请基于现在的 finance 台账，给我一版本周财务整理简报。

重点写：
1. 本周新增了哪些支出
2. 大概的分类结构
3. 哪些还缺发票
4. 哪些记录还缺关键信息
5. 我下一步最该先补什么
```

这一步的目标不是“文风好不好”，而是：

- 简报有没有真的建立在当前台账之上
- 下一步行动项是不是清楚

## 8. 如果第一轮表现不好，优先看什么

如果结果不理想，优先看这几层：

1. 它是不是直接脑补了缺字段
2. 它是不是没有把结构整理出来
3. 它是不是没真正写回工作区
4. 它是不是没把缺票和待确认项单独列出
5. 你给它的字段约定是不是还不够明确

## 9. 什么时候适合继续留在主 Agent

如果你现在只是：

- 个人使用
- 低频整理
- 月底偶尔汇总

那这条线完全可以先留在 `main` 里验证。

## 10. 什么时候该拆成专用财务 agent

如果开始出现下面这些情况，就更适合拆成专用财务 agent：

1. 财务资料明显敏感
2. 你已经形成固定台账结构
3. 你要单独 reminders 或 cron
4. 你要长期做回款 / 付款跟进
5. 你不希望其他任务污染财务上下文

## 11. 第二阶段最值得继续做什么

等这轮通过后，第二阶段最值得继续补的是：

- 回款与付款提醒
- 缺票自动周提醒
- 月底经营简报
- 飞书多维表格版财务台账

其中优先级最高的是：

**回款与付款提醒**

因为它最能把：

- 财务台账
- 提醒
- 周期复盘

三件事真正接起来。
