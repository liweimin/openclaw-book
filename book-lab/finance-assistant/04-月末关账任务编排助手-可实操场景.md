# 04-月末关账任务编排助手：可实操的财务落地场景

## 为什么选这个场景

在培训文档里列出的 8 个财务场景中，我们已经用 03 篇教程走通了"报销台账"。
现在我们要来一个**真正能让财务总监眼前一亮**的场景：

> **月末关账任务编排助手**
> 让 AI 自动提醒各模块负责人完成关账动作，收集完成证据，自动汇总"卡点清单"，并在飞书里形成每天的关账进度日报。

这个场景之所以极其适合演示和写进书里，原因有三：
1. **痛点真实**：每个月末，财务团队最头疼的就是催人、催表、催确认，全靠人肉群发消息和手动对账。
2. **能力全覆盖**：它同时用到了 OpenClaw 的 **cron 定时**、**飞书消息**、**多会话追问**、**工作区文件落盘**、**记忆沉淀** 五大核心能力。
3. **WSL 可跑通**：不依赖外部 ERP 或银行系统，只需要飞书群 + 本地文件，你一个人就能模拟整套流程。

---

## 第〇步：理解这个场景的完整闭环

```
关账日历（cron 定时触发）
      ↓
Agent 按日历向飞书群/私聊发出提醒
      ↓
各负责人在飞书回复"已完成"/"未完成 + 原因"
      ↓
Agent 自动更新工作区里的关账进度表
      ↓
Agent 每天定时汇总，生成"关账日报"发到飞书群
      ↓
对未完成项自动追问催办
```

你在演示时，可以一个人扮演 3 个角色（收入会计、费用会计、总账会计），分别在飞书里跟 Agent 互动，模拟真实的月结流程。

---

## 第一步：创建关账专用 Agent

打开 WSL 终端：

```bash
# 确保飞书插件已安装（如果之前装过就跳过）
openclaw plugins install @openclaw/feishu

# 创建专属关账助手（注意：命令是 agents add，不是 create）
openclaw agents add closing-bot --workspace /mnt/d/openclaw-workspace/closing-bot
```

---

## 第二步：编写关账助手的配置文件

打开 `~/.openclaw/openclaw.json`，在 `agents` 段里找到刚才自动生成的 `closing-bot` 块，补全配置：

```json5
{
  agents: {
    "closing-bot": {
      // 工作区（agents add 时已自动填入）
      workspace: "/mnt/d/openclaw-workspace/closing-bot",

      // 大模型
      model: {
        primary: "openai/gpt-5.2"  // 用你实际可用的模型
      },

      // 人设
      persona: "你是集团财务部的月末关账编排助手。你的职责是：1. 按照关账日历提醒负责人完成关账动作；2. 收到回复后判断已完成或未完成并记录原因；3. 每天汇总关账进度生成日报；4. 绝不替任何人做财务判断，只做任务追踪和信息归集；5. 所有进度更新必须写入工作区的 closing-tracker.md。",

      // 记忆
      memorySearch: {
        provider: "openai",
        model: "text-embedding-3-small",
        remote: {
          apiKey: "YOUR_OPENAI_API_KEY"
        }
      }
    }
  },

  // 飞书渠道（全局共享配置）
  channels: {
    feishu: {
      enabled: true,
      connectionMode: "websocket",
      dmPolicy: "pairing",
      groupPolicy: "allowlist",
      groupAllowFrom: ["oc_closing_group"],  // 下面会教你怎么拿到这个 ID
      groups: {
        "oc_closing_group": {
          requireMention: true,
          replyInThread: "enabled",
          groupSessionScope: "group_topic_sender"
        }
      },
      tools: {
        chat: true,
        doc: true,
        wiki: false,
        drive: false,
        scopes: true
      }
    }
  }
}
```

> **注意**：不需要单独建 `closing-bot.json` 文件，所有 Agent 都在同一个 `openclaw.json` 里管理。

## 第三步：准备关账日历模板（工作区文件）

在工作区目录下创建关账任务清单文件。

```bash
# 创建工作区目录
mkdir -p /mnt/d/openclaw-workspace/closing-bot
```

## 第四步：在飞书里建立关账专用群

1. 在飞书里新建一个群，命名为 `【财务】3月关账协调群`。
2. 把你的关账机器人拉进群。
3. 在群里 @机器人 随便发一句话。
4. 回到 WSL 终端，执行：
```bash
openclaw logs --follow
```
5. 在日志里找到类似 `chat_id: "oc_xxxxxxxx"` 的字样，把这个 ID 复制出来。
6. 把这个 ID 填回上面配置文件里的 `groupAllowFrom` 和 `groups` 对应位置。

---

## 第五步：启动服务并绑定飞书

```bash
# 如果之前没绑定过飞书凭证，先执行：
openclaw channels add
# 选择 feishu，按提示输入 App ID 和 App Secret

# 把 closing-bot 绑定到飞书渠道
openclaw agents bind --agent closing-bot --bind feishu

# 重启 Gateway 让所有配置生效（注意：没有 --agent 参数）
openclaw gateway restart

# 开另一个终端窗口监控日志
openclaw logs --follow
```

## 第六步：设置定时提醒（cron）

这是这个场景的灵魂！我们要让 Agent 每天早上 9 点自动检查关账日历，并向飞书群发出当天的任务提醒。

在飞书私聊中（或终端里），对 `closing-bot` 说：

```text
请帮我设置一个定时任务：
每天早上 9:00 自动执行以下动作：
1. 读取工作区里的 closing-calendar.md
2. 找出今天应该提醒的任务
3. 在飞书关账协调群里 @ 对应负责人，提醒他们今天需要完成的关账动作
4. 如果有昨天标记为"未完成"的任务，也一并催办

请用 cron 表达式 "0 9 * * *" 来设置。
```

Agent 会帮你调用 `cron` 工具创建定时任务。

> **WSL 避坑提醒**：cron 任务依赖 Gateway 进程保持运行。如果你的 WSL 关机了或 Gateway 挂了，定时任务会失效。演示时请确保 Gateway 一直在前台跑着。

## 第七步：模拟关账互动（一个人扮演多个角色）

现在是最精彩的环节！你可以一个人在飞书群里模拟整个关账流程。

### 场景演绎 A：收入会计回复已完成

在飞书关账群里用你自己的账号发：
```text
@关账助手 任务1已完成，所有应收发票已开具，共计 47 张，金额合计 ¥2,340,500。
```

**预期效果**：Agent 会：
1. 在飞书群里确认收到
2. 自动更新工作区的 `closing-tracker.md`，把任务1标记为"已完成"
3. 记录完成时间和关键数据

### 场景演绎 B：费用会计回复未完成

换个角度（你可以改个飞书昵称，或直接用另一句话模拟）：
```text
@关账助手 任务3还没做完，还有3张供应商发票没收到，预计明天下午能补齐。
```

**预期效果**：Agent 会：
1. 把任务3标记为"未完成"
2. 记录原因："3张供应商发票未收到"
3. 记录预计完成时间："明天下午"
4. 在 `closing-tracker.md` 里更新，并标注为红色风险项

### 场景演绎 C：触发关账日报

在飞书私聊中对 Agent 说：
```text
现在帮我生成今天的关账进度日报，发到关账协调群里。
```

**预期效果**：Agent 会读取 `closing-tracker.md`，并在飞书群里发出类似：

> 📊 **3月26日关账进度日报**
>
> ✅ 已完成：1/3
> - [✅] 任务1：应收发票开具（张三）- 已完成，47张，¥2,340,500
>
> ⚠️ 进行中：1/3
> - [⏳] 任务3：费用发票入账（王五）- 未完成，3张供应商发票待补，预计明天下午
>
> ❌ 未开始：1/3
> - [❌] 任务2：银行余额调节表（李四）- 尚未回复
>
> 📌 **今日重点跟进**：请李四尽快确认银行余额调节表进度。

## 后续可扩展方向

当这个基础版跑通后，后续可以继续加码：

1. **飞书文档版日报**：不只是群消息，而是自动生成一份飞书文档，带表格和颜色标记。
2. **多 Agent 协同**：给每个会计模块分配一个专属子 Agent，主 Agent 只做汇总调度。
3. **关账历史对比**：利用 memory，让 Agent 记住上个月各任务的完成时间，本月自动对比是否延迟。
4. **审计证据包**：关账结束后，自动把所有确认记录、附件、截图打包成一个审计交付目录。
