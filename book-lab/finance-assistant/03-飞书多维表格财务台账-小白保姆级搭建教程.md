# 03-飞书多维表格财务台账：小白保姆级搭建教程
（结合 WSL/本地部署的避坑指南）

前两篇文档《蓝图》和《验证手册》讲的是基于本地 Markdown 文件的基本打雷逻辑。而在真实的财务、报销和台账场景里，**「飞书多维表格 (Bitable)」** 才是真正的王炸大杀器！

很多小白朋友看完前面可能会一头雾水：“我到底该在命令行敲什么？怎么建机器人？飞书怎么连？”

这篇保姆级教程，将**从零开始，一行行命令**手把手带你完成：
**把 OpenClaw 变成你的私人财务出纳，你在微信/飞书随口说一句花销，它自动帮你整理成标准格式，并实时写入飞书多维表格。**

---

## 零、前期准备工作：装插件与建 Agent

我们不建议用默认的全局大管家来管钱，所以我们要专门建一个“财务小助理”。
打开你的本地终端（或者 WSL 命令行），一行行执行：

### 1. 安装飞书基建插件
这个插件让 OpenClaw 拥有连接飞书和操作多维表格的超能力：
```bash
openclaw plugins install @openclaw/feishu
```
*(如果你是用源码跑的，可以执行 `openclaw plugins install ./extensions/feishu`)*

### 2. 创建一个专属的财务 Agent
不要用默认配置，我们给它个编制：
```bash
# 创建专属财务 Agent，同时指定它独立的工作区目录
openclaw agents add finance-bot --workspace /mnt/d/openclaw-workspace/finance-bot
```
> **注意**：命令是 `agents add`，不是 `agents create`！
> 创建后，它会自动在 `openclaw.json` 的 `agents` 段落里新增一个 `finance-bot` 配置块。

### 3. 提前建好工作区目录
```bash
mkdir -p /mnt/d/openclaw-workspace/finance-bot/finance/ledger
mkdir -p /mnt/d/openclaw-workspace/finance-bot/finance/invoices
```

## 第一步：准备一个飞书多维表格 (Bitable)

我们需要先给 Agent 准备一个“账本”。

1. 打开飞书云盘，点击 **[新建] -> [多维表格]**。
2. 命名为：`2026 个人财务报销台账` （名字你可以随便取）。
3. **关键操作：建好规范的表头（字段）**。请严格按照以下名称和类型建立列（这也是为了配合 Agent 读写）：
   *   **费用发生日期** `[日期型]`：如果是当天的支出，Agent 也会自动填今天。
   *   **金额 (元)** `[数字型]`：填纯数字。
   *   **支出类目** `[单选标签]`：提前在这列设置好几个标签（打车差旅、餐饮招待、购买软件、办公用品、其他）。
   *   **付款对象说明** `[多行文本]`：记录付给谁或是干嘛的。
   *   **关联项目方** `[多行文本]`：例如华东二期项目、某某客户。
   *   **是否可报销** `[单选标签]`：标签设为（是、否、待定）。
   *   **发票状态** `[单选标签]`：标签设为（已开票、未开票、不需要开票）。
   *   **备注** `[多行文本]`：放一些乱七八糟的琐事说明。

*(建好之后，你可以随便填一行假数据作为第一排，方便 Agent 等会儿“认路”。)*

---

## 第二步：在飞书后台创建机器人生命并给足权限

OpenClaw 只是灵魂，我们要在飞书里给他制造一个肉体（App）。

1. 登录飞书开放平台开发者后台 (https://open.feishu.cn/app/)。
2. 点击 **[创建应用] -> [企业自建应用]**，名字就叫 `我的财务出纳 Agent`。
3. **开启机器人**：在左侧菜单点击 [添加应用能力]，把 [机器人] 那个卡片开启。
4. **长连接收发消息**：在左侧菜单点击 [事件与回调]，选择“长连接（WebSocket）”，并添加 `im.message.receive_v1` 事件。
5. **极其重要的权限管理**：在左侧菜单找到 **[权限管理]**。搜索以下权限，并全部点击 **[开通]**：
   *   `获取多维表格内容` (bitable:app:readonly)
   *   `查看、评论、编辑和管理多维表格` (bitable:app)
   *   `获取云空间文件和文件夹信息` (drive:drive:readonly)
   *   `查看、评论、上传、管理云空间的文件/文件夹` (drive:drive)
6. **拿钥匙与发版**：
   *   去左侧菜单点击 **[凭证与基础信息]**，把 `App ID` 和 `App Secret` 复制下来，一会要用。
   *   去左侧菜单点击 **[版本管理与发布]**，**重新创建一个版本并发布审核**（如果在企业内通常免审直接秒过）。如果不发新版本，刚刚开的权限都是无效的！

## 第三步：将 OpenClaw 与飞书打通并绑死配置

现在，我们要把刚才复制的 `App ID`（钥匙）告诉本地的 Agent。

### 1. 录入飞书通行证
回到本地终端，执行：
```bash
openclaw channels add feishu
```
系统会弹出行提示，让你分别输入刚才在飞书后台拿到的 `App ID` 和 `App Secret`。按照提示贴进去就行。

### 2. 给 finance-bot 专属 Agent 配置飞书能力
我们需要编辑 `openclaw.json` 配置文件（通常在 `~/.openclaw/openclaw.json`），在 `agents` 段里找到刚才自动生成的 `finance-bot` 块，把飞书相关配置补全：

```json5
{
  agents: {
    "finance-bot": {
      // 工作区（刚才 agents add 时已自动填入）
      workspace: "/mnt/d/openclaw-workspace/finance-bot",

      // 大模型（用你实际可用的）
      model: {
        primary: "openai/gpt-5.2"
      },

      // 人设：让它像个专业会计
      persona: "你是一个极其严谨的财务报销出纳。如果发现任何账单缺少信息（如：没说项目、没说发票状态），你绝不脑补，必须主动追问用户。确认无误后，你负责调用系统工具将数据规范地录入飞书多维表格台账中。",

      // 记忆配置（让它记住你的偏好和历史台账）
      memorySearch: {
        provider: "openai",
        model: "text-embedding-3-small",
        remote: {
          apiKey: "YOUR_OPENAI_API_KEY"
        }
      }
    }
  },

  // 飞书渠道配置（这是全局的，所有 agent 共享）
  channels: {
    feishu: {
      enabled: true,
      connectionMode: "websocket",
      dmPolicy: "pairing",
      tools: {
        chat: true,
        doc: true,
        wiki: true,
        drive: true,
        scopes: true
      }
    }
  }
}
```

> **配置结构说明**：
> - `agents.finance-bot` 块里放 Agent 私有的配置（工作区、人设、模型）
> - `channels.feishu` 块是全局共享的飞书连接和权限配置
> - 你不需要单独建一个 `finance-bot.json` —— 所有 Agent 都在同一个 `openclaw.json` 里管理

### 3. 把 finance-bot 绑定到飞书渠道
```bash
# 把 finance-bot 绑定到飞书私聊入口
openclaw agents bind --agent finance-bot --bind feishu
```

### 4. 启用记忆钩子并启动服务
```bash
# 启用会话记忆（这样 /new 时会自动沉淀对话摘要）
openclaw hooks enable session-memory

# 重启 Gateway 让所有配置生效
openclaw gateway restart
```

> **WSL 用户避坑指南：**
> 如果你在 WSL 跑服务，而 Windows 跑着飞书客户端：
> 请再新开一个 WSL 窗口运行 `openclaw logs --follow` 实时监控日志。WSL 极其容易在 Windows 锁屏休眠时把本地网络层休克，如果你发给机器人的消息没反应，盯着日志执行 `openclaw gateway restart` 重启一下。

## 第四步：实战！让 Agent 开始录入账本

打开飞书，找到你创建的那个“我的财务出纳 Agent”，开启私聊。

（因为你配置了 `dmPolicy: "pairing"`，第一次发消息它会给你一个验证码，你在 WSL 原本那个终端里输入 `openclaw pairing approve feishu xxxxxx` 验证身份后就能畅聊了）

### 阶段 1：先让它认路读表

**在飞书对话框发送：**
> “你好，我刚刚建了一个新的财务台账。这是链接：[粘贴你刚才那页多维表格的分享链接地址]。请你先只读取这里的表结构和字段信息，告诉我你看懂了没，**不要擅自添加任何数据**。”

**预期效果：**
稍等大概 10~20 秒，你应该会看到终端跳过一长串底层 JSON 通信日志，然后它在飞书会回答：
> “我已经成功读取了多维表格信息。这似乎是一个财务报销台账，一共有 [日期、金额、支出类目...] 等 8 个字段。我准备好了。”

### 阶段 2：模糊输入，让它追问和规范化

测试它是不是一个好会计（好会计不能乱记账）。

**在飞书发送模糊消费：**
> “昨天晚上请王总吃饭花了 640 元。”

**预期效果：**
它不应该直接写进去（因为信息太少）。它应该基于表头结构回复你类似：
> “好的，已记录昨天的餐饮消费 640 元。但为了规范写入台账，我还缺少一些信息：
> 1. 这个消费归属在哪个关联项目方里？
> 2. 是否可报销？
> 3. 目前发票开好了吗？
> 请补充告诉我，我再帮您正式写入多维表格。”

### 阶段 3：补充信息，一锤定音落盘

**在飞书对它回复：**
> “项目是华东渠道维护，肯定能报销，但饭店说下个月统一给我开票所以现在没票。”

**预期效果：**
Agent 会在后台自动调用底层 `insert_record` 工具。然后告诉你：
> “操作完成！已经将这条记录成功写入到了《2026 个人财务报销台账》中：
> - 费用发生日期：2026-03-10（根据您的‘昨天’推算）
> - 金额：640
> - 支出类目：餐饮招待
> - 付款对象说明：请王总吃饭
> - 关联项目方：华东渠道维护
> - 是否可报销：是
> - 发票状态：未开票”

**最后一步：**
立刻打开你的飞书多维表格（Bitable）。你会发现，刚才这个数据已经**真真切切地出现在了绿油油的表格里面！**

## 第五步：高级玩法（随时随地用嘴查账对账）

当一个月结束，这上面已经密密麻麻记录了五十多笔打车、吃饭的账务。

**不用打开电脑去跑 Excel 算求和公式，你直接在下班路上，掏出手机飞书对机器人说：**
> “帮我查一下，上个月这个多维表格 [附链接] 里面，状态还是**【未开票】**而且**【是否可报销 是】**的所有记录，帮我汇总成一个清单发给我，并计算一下我总共还没拿到发票的钱有多少？”

Agent 会立刻检索那份上千行的表格，做过滤筛选、做数学汇总，最后在飞书上丢给你：
> “您上个月共有 4 笔可报销但未拿发票的款项：
> 1. 3月10日 请王总吃饭 640元
> 2. 3月14日 上海出差打车 120元...
> 未开票待追讨的总报销额为：2140 元。建议您尽快联系上述商户。”
