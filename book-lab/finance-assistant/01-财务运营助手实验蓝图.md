# 财务运营助手实验蓝图

这份蓝图不是正式书稿章节，而是“财务运营助手”这条实验线的第一版设计稿。

它要回答的是：

- OpenClaw 在财务相关场景里，最适合先做什么
- 为什么不应该一开始就做高风险投资或税务顾问
- 财务助手第一条最稳的落地链路是什么
- 什么时候继续用主 Agent，什么时候拆专用财务 agent

## 1. 这条实验要做成什么

这条实验想做的不是：

- 一个会给投资意见的财经机器人
- 一个替代财务负责人的“万能助手”

而是：

**一个能帮个人或小团队整理财务运营动作的助手。**

第一版最稳的落地方向是：

1. 收集零散收支与报销信息
2. 整理成统一台账
3. 标出待补信息和缺发票项
4. 输出一版本周 / 本月财务整理结果
5. 后续再进入提醒与经营简报

## 2. 为什么不先做投资、税务、法律方向

原因很直接：

- 风险高
- 时效强
- 对外部事实依赖重
- 错误成本高

而“财务运营整理”这条线更适合先做，因为它更像：

- 文件整理
- 信息结构化
- 台账维护
- 提醒与复盘

这些正好是 OpenClaw 更擅长、也更容易验证的事情。

## 3. 第一版最值得先做什么

第一版建议先做：

**报销与发票台账助手**

因为它同时满足：

- 高频
- 刚需
- 规则清晰
- 容易验证“有没有真正落盘”
- 很适合后面再接提醒

## 4. 这条助手到底解决什么问题

很多个人创业者、小团队负责人、项目负责人，现实里经常会遇到这些问题：

- 报销材料散在聊天、截图、便签和文件夹里
- 记得有支出，但忘了有没有票
- 到月底才发现很多信息没记全
- 想看本月支出结构时，要自己重新翻聊天
- 想知道哪些款该付、哪些款该催，但没有清楚台账

这个助手的目标不是替你做财务判断，而是帮你把这些动作收拢成稳定流程。

## 5. 第一版默认范围

第一版先收紧范围，只验证：

1. 费用记录收集
2. 台账整理
3. 发票状态追踪
4. 周 / 月汇总

先不急着验证：

- 对外部财务系统深度集成
- 自动税务判断
- 自动会计分录
- 非常复杂的预算模型

## 6. 推荐的实验路线

### 阶段 1：先把零散信息整理成统一记录

第一阶段重点看：

- 它能不能把聊天里的零散费用信息整理成统一格式
- 缺字段时会不会主动追问

### 阶段 2：再把记录落到台账

这一阶段重点看：

- 能不能写回工作区
- 写入结构是不是长期可维护

### 阶段 3：再做缺票、待报销、异常项

这一阶段重点看：

- 它能不能从已记录数据里抽出待补项
- 能不能形成后续跟进清单

### 阶段 4：最后再做提醒与财务简报

等前面稳定后，再进入：

- 每周提醒补票
- 月底整理
- 经营简报
- 回款 / 付款提醒

## 7. 第一版推荐的字段

第一版台账建议至少固定下面这些字段：

1. 日期
2. 金额
3. 类别
4. 付款对象
5. 关联项目 / 客户
6. 是否可报销
7. 发票状态
8. 备注

这 8 个字段已经足够支撑第一轮验证。

## 8. 第一版推荐的工作区结构

第一版不用上来就做复杂系统，建议先这样：

```text
finance/
├─ inbox/
│  └─ raw-notes.md
├─ ledger/
│  └─ 2026-03.md
├─ invoices/
│  └─ missing.md
└─ summaries/
   └─ 2026-03-week2.md
```

最简单的理解是：

- `inbox/`：先收集原始材料
- `ledger/`：放结构化台账
- `invoices/`：放缺票与待补项
- `summaries/`：放阶段性汇总

## 9. 先用主 Agent 还是先拆财务 agent

这条线分两种情况：

### 情况 A：个人或极轻量小团队

可以先用默认主 Agent 验证。

适合：

- 你只是想先验证流程能不能跑
- 财务资料量不大
- 暂时没有多人协作和高频提醒

### 情况 B：开始长期化或敏感化

更适合拆成专用财务 agent。

适合：

- 资料明显敏感
- 要独立工作区
- 要专门的提醒节奏
- 要长期做回款 / 付款跟踪

## 10. 第一轮成功标准

第一轮实验通过，至少要满足：

- 它能把零散财务记录整理成统一结构
- 缺字段时会主动追问
- 能把结果真正写回工作区
- 能单独列出待补发票和待确认项
- 能输出一版周 / 月整理简报

## 11. 后面最值得长成的子助手

如果第一轮通过，这条线后面最适合继续长成：

1. 报销与发票台账助手
2. 回款与付款提醒助手
3. 月度经营简报助手
4. 预算偏差跟踪助手

其中最建议优先继续做的是：

**回款与付款提醒助手**

因为它最能体现 OpenClaw 在：

- 台账
- 提醒
- 周期复盘

这三层能力上的组合价值。
