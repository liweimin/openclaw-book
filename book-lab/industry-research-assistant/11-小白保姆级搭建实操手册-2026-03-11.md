# 行业研究助手：小白保姆级搭建实操手册

看完前面的“理发师”设计思路，是不是想亲手捏一个听话的行业研究助手了？

这份手册就是为你准备的。不用懂代码，只要跟着下面的 4 步走，15分钟内，你就能在自己的电脑（WSL）上搭出一个“先问需求、再做研究、不瞎编连网”的专属助理。

---

## 第一步：建一个干干净净的“独立助理”

行业研究和帮你安排日程是截然不同的两件事。为了不串台，我们要给它单开一个独立的房间。

打开你的 WSL 命令行终端，输入下面这行命令：

```bash
openclaw agents add industry-research-assistant --workspace ~/.openclaw/workspace-industry-research-assistant
```

**这一步做完后发生了什么？**
OpenClaw 帮你在 `~/.openclaw/agents/` 下面建了一个名叫 `industry-research-assistant` 的文件夹。而且它有自己专属的“工作区”（用来存未来写出的行业报告和记忆），完全不会把你个人的隐私日程弄乱。

---

## 第二步：为它穿上“神装”（配置兜底工具包）

前面的思路篇我们讲过，查资料最怕网站“反爬”或者突然打不开。所以我们必须给它装上“备胎”（Fallback Skills）。这样它在默认的 `web_fetch` 碰壁时，能自己切换武器继续挖。

连续输入下面这 4 行命令，给你的助理装上完整的“探测器套件”：

```bash
# 备用搜索引擎
openclaw skills install tavily-search --agent industry-research-assistant

# 强力网页内容提取器（对付难啃的网页）
openclaw skills install firecrawl-search --agent industry-research-assistant

# 复杂网页模拟器（需要真正的浏览器点击时）
openclaw skills install agent-browser --agent industry-research-assistant

# 长文摘要压缩器（读几万字的长报告时用来提炼）
openclaw skills install summarize --agent industry-research-assistant
```

装完以后，记得去 `openclaw configure --section web` 里面，确保你填入了真实的 `TAVILY_API_KEY` 和 `FIRECRAWL_API_KEY`。

---

## 第三步：注入“理发师灵魂”（配置 Persona）

为什么有的 AI 直接给你扔废话，有的却会像真人一样追问？秘密就在于它的“出厂人设”。

我们要建一个 `persona.md` 文件，明确告诉它规矩。

**操作步骤：**
用你的代码编辑器或者命令行，在 `~/.openclaw/agents/industry-research-assistant/` 这个文件夹下，新建一个文件叫 `persona.md`。

把下面这段最重要的“规矩”直接复制粘贴进去并保存：

```markdown
你是一个专业的“行业研究助手”。
你的核心规矩是：不要在信息明显不足时直接开始全网检索。

当用户提出行业研究需求时，你必须先确认下面这张“任务卡”：
1. 研究对象与细分赛道（是中国还是全球？是整个食品还是只专注调味品？）
2. 目标读者是谁？关注哪些维度？
3. 有没有指定的偏好来源？或必须避开的营销号？

**必须的动作：**
1. 收到需求后，先向用户复述你理解的任务卡，如果有缺失，向用户追问最多3个核心问题。
2. 必须等用户确认了再开始干活。
3. 干活时：优先用内置 `web_search` 和 `web_fetch` 寻找官方和一手来源（不仅限于搜索，必须点开链接阅读原文）。
4. 如果内置读取工具被拦截失败，不要直接放弃，立刻使用你安装的 `firecrawl-search` 或 `agent-browser` 作为备用抓取方案。
5. 在最终交付的报告结尾，必须附带你“实际搜索过的关键词”和“实际阅读过的来源链接”。
```

这个文件保存好之后，你的助手就有了“不瞎猜、听人劝、带出处”的灵魂。

---

## 第四步：在飞书里的“首次相亲”

在 WSL 重启一次网关，让上面的配置生效：
```bash
openclaw gateway restart
```

现在，打开你的飞书私聊框（或者你绑定了这个 Agent 的那个专门的机器人会话）。

发第一句话测试它：
> **“我想把你当成一个通用的行业研究助手来用，帮我看看调味品行业最近怎么样。但请先不要直接开始搜索，先帮我把研究简报问清楚再确认。”**

**怎么判断你搭建成功了？**
如果它**没有**立刻长篇大论给你讲调味品历史，而是回复你类似这样的话：
> *“好的，在开始研究前，为了确保方向准确，我想向您确认几个关键细节：您指的地域是中国还是全球？您重点关注的是销量、政策还是融资并购？……”*

**恭喜你！**
最完美的第一步已经走通了。它已经不再是一个傻傻的搜题神器，而是一个懂得先拿需求卡对齐的合格助理。

---

## 后续进阶路线：让它成为真正的私人管家

等你和它把“一次性研究”的磨合做得足够顺畅后，你接下来可以玩更高级的自动化操作。

### 进阶 1：给它装上“工作区大脑”（文件读写能力）

如果你想让它记住你关注的东西，或者帮你把报告整理成专门的 Markdown 文件，你需要给它开启读写本地文件的能力。

**操作步骤：**
你需要修改 OpenClaw 的总配置文件 `~/.openclaw/openclaw.json`，在这个助手的专属配置里，增加一行工具授权：

```json5
// 找到 agents.list 里的 industry-research-assistant，修改 tools.alsoAllow：
{
  "id": "industry-research-assistant",
  "tools": {
    "profile": "messaging",
    "alsoAllow": ["web_search", "web_fetch", "browser", "read", "write"] // 加上 read 和 write
  }
}
```

配置好并重启网关后，你可以在飞书里直接命令它：
> **“把今天的调研结论和我们确认的研究偏好，写进工作区 root 目录下的 `daily-report.md` 文件里。”**

它就会自动把排版精美的研报文件存在你的电脑上。下次你问到相关内容，它还可以去读取这个文件帮你做知识梳理。

### 进阶 2：解锁“按时打卡”（建立 Cron 定时任务）

每天在飞书里问它同样的问题太麻烦了。一旦你把它的查资料手法调教成熟了，就可以让它每天定时自动给你干活。

> **提示：在建立定时任务前，一定要确保它已经在前一天手动帮你出过一份你非常满意的研报。**

**操作步骤：**
在工作日，如果你觉得刚才那份调味品报告非常棒，你直接在飞书里对它说：
> **“既然你已经记住了我的偏好和抓取逻辑，请帮我建立一个定时任务（cron）。从明天开始，每个工作日早上 8:30，自动按今天的标准给我出具并推送《调味品行业每日追踪简报》。”**

OpenClaw 聪明的地方在于，它会把如何配置 Cron 任务转换成它自己的后台指令。你只需要像老板一样对它提要求，它会向你展示一个类似下面这样的 Cron 表达式：
`30 8 * * 1-5` （每周一到周五上午 8:30）

确认无误后，你就可以彻底放手了。明天早上 8:30 坐在工位上，打开飞书，你的专属行业研究助理就已经把热腾腾的追踪简报发给你了。

---

**最后再强调一句：**
搭 AI 助理就像培养实习生。不要指望第一天刚认识，它就能完美解决你的所有问题。
先把 **前四步基础篇** 练熟，确保它真的学会了“找对来源、说真话不捏造”，再慢慢开始教它写文件和定时推送。

现在，快去 WSL 里建好你的代理吧！！
