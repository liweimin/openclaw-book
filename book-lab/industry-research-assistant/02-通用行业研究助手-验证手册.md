# 通用行业研究助手验证手册

这份手册服务的是第一轮实验验证。

目标不是一下把所有能力做满，而是先把下面这条链路跑通：

`飞书发起研究 -> 先问清研究简报 -> 固定研究任务 -> 输出一版结果 -> 写回工作区`

## 1. 第一轮验证范围

第一轮先验证：

- 单独 agent
- 飞书私聊入口
- Skill 正常触发
- 第一次追问是否足够好
- 能否形成清晰研究简报
- 能否基于研究简报跑出首版结果
- 能否把结果沉淀到工作区

第一轮先不强求：

- 日更 cron
- 周报归档
- 飞书文档自动写入
- 很复杂的多来源长期记忆

## 2. 推荐的 agent 设计

建议单独建一个 agent，例如：

```text
industry-research-assistant
```

不要和 `weekly-assistant` 混在一起。

## 3. 第一轮验证的固定目标

第一轮先不把行业写死。

这一轮真正要验证的是：

- 它能不能把一个模糊的行业研究需求，收敛成一份清晰研究简报
- 它能不能先问关键问题，而不是直接开始搜
- 它能不能把“通用助手”用到具体行业上

## 4. 第一轮飞书验证怎么发

第一轮建议你直接在飞书里发这样一段：

```text
我想把你当成一个通用的行业研究助手来用。

请先不要直接开始搜索，先帮我把研究简报问清楚并确认。

我希望你第一次先把一个行业研究任务定义清楚，而不是直接开搜。

请至少帮我确认这些内容：
1. 研究对象和细分赛道
2. 地域范围
3. 目标读者
4. 关注维度
5. 重点公司、品牌、渠道、产品或关键词
6. 优先来源与禁用来源
7. 输出频率
8. 输出形式和投递位置

你先做两件事：
1. 用 4 到 8 条要点复述你理解到的研究简报
2. 再问我最多 5 个关键问题，帮助你把第一版研究任务定清楚

不要一上来直接给行业结论。
```

## 5. 第一轮通过标准

如果下面这几件事都发生了，说明第一轮方向是对的：

- 它先复述理解，而不是直接开搜
- 它问的问题和研究简报强相关
- 它没有忽略目标读者与输出频率
- 它把“研究简报”理解成一张任务卡，而不是泛泛聊天

## 6. 第二轮验证：用一个具体行业去跑

等你回答完它的追问后，再拿一个具体行业去跑。

建议先用“调味品”作为第一个实例：

```text
现在请基于我们刚才确认好的研究简报，开始做第一版调味品行业趋势研究。

要求：
1. 优先使用官方、协会、公司官网、财报、IR 等一手来源
2. 再补行业媒体
3. 如果某些维度今天没有明显变化，可以不强行写
4. 输出给公司一把手看，所以先结论、后事实、再来源

请按这个结构输出：
1. 今日最重要的 3 个变化
2. 重点公司/品牌动态
3. 趋势与风险提醒
4. 今天值得继续追的 3 个问题
5. 参考来源
```

## 7. 第三轮验证：写回工作区

如果它输出的首版研究结果基本可用，再让它把“研究简报 + 首版结果”一起写回工作区。

建议用这种明确说法：

```text
请把今天确认过的研究简报和今天的首版研究结果写回工作区。

至少写到下面两类文件：
1. 研究画像文件
2. 今天的研究结果文件

写完后告诉我：
1. 写了哪些文件
2. 每个文件里写的是什么
```

## 8. 第一轮不通过时，先看什么

如果第一轮表现不好，优先看这几层：

1. 它是不是没先问清研究简报
2. 它是不是忽略了目标读者
3. 它是不是来源优先级不对
4. 它是不是没有真的调用 browser / 读网页
5. 它是不是没有把结果写回工作区

## 9. 第二阶段再做什么

等第一轮通过后，第二阶段再进入：

- 调味品日报 / 周报
- OpenClaw / Agent 产品赛道跟踪
- cron
- 重点来源清单沉淀
- 研究画像长期维护
