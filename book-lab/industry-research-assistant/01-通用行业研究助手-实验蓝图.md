# 通用行业研究助手实验蓝图

这份蓝图不是正式书稿章节，而是“行业研究助手”这条实验线的第一版通用设计稿。

它要回答的是：

- 这条实验到底要做成什么
- 为什么第一版先做成“通用助手”，而不是某个行业专用助手
- 第一次对话到底要固定哪些字段
- 后续怎么从一次性研究升级成持续跟踪

## 1. 这条实验要做成什么

这条实验想做的不是“调味品助手”或“OpenClaw 助手”这样的单行业 Demo，而是：

`一个通用的行业研究助手`

它的工作方式应该是：

1. 第一次对话先把研究任务问清楚
2. 形成一份稳定的“研究简报”
3. 再基于这份研究简报去检索、整理、输出
4. 后续再按日报、周报、月报去持续跟踪

也就是说，它先解决的是：

`研究任务定义清不清楚`

而不是：

`某一个行业今天写得漂不漂亮`

## 2. 为什么要先做通用，不先做专用

用户当前同时有两个真实方向：

- 调味品
- OpenClaw

如果第一版把助手写死成“调味品专用”，后面再做 OpenClaw、AI Agent、别的消费赛道时，就得重新搭一套。

更好的做法是：

- 先把通用框架做对
- 再把调味品当成第一个案例
- 后面再把 OpenClaw 当成第二个案例

这样这条实验线才真正有复用价值。

## 3. 第一次对话要固定什么

第一次对话的目标，不是直接开始搜，而是先固定一份研究简报。

建议第一版至少固定下面这些字段：

1. 研究对象
2. 细分赛道
3. 地域范围
4. 目标读者
5. 关注维度
6. 重点公司 / 品牌 / 渠道 / 产品
7. 优先来源
8. 禁用来源
9. 优先链接
10. 输出频率
11. 输出形式
12. 投递位置

可以把它理解成一张“研究任务配置卡”。

## 4. 这条实验的第一版默认原则

第一版通用助手默认遵守这些原则：

- 不在信息不足时直接开搜
- 先复述自己理解到的研究简报
- 缺关键字段时先追问
- 用户给了具体链接、网站、来源优先级时必须尊重
- 输出必须和目标读者匹配

## 5. 先验证什么

第一版先验证 4 件事：

1. 第一次追问是否足够好
2. 能不能把模糊研究需求收敛成一份清晰研究简报
3. 基于这份简报做出的第一版研究是否靠谱
4. 结果能不能沉淀到工作区，供后续持续跟踪使用

先不急着验证：

- 多行业同时跑
- 很复杂的 cron 编排
- 自动飞书文档成文
- 太重的长期记忆体系

## 6. 推荐的实验路线

### 阶段 1：验证“第一次询问”

第一轮最重要的不是结果，而是第一次询问。

要先验证：

- 它会不会先问清楚
- 问的问题够不够关键
- 能不能形成研究简报

### 阶段 2：验证“一次性研究”

等研究简报稳定后，再验证：

- 检索顺序是否合理
- 来源优先级是否正确
- 输出是否适合目标读者

### 阶段 3：验证“持续跟踪”

等一次性研究稳定后，再进入：

- 日报
- 周报
- 月报
- cron
- 工作区持续沉淀

## 7. 推荐的最小技术路径

这条实验默认走下面这套最稳路径：

- 单独 agent
- 飞书私聊入口
- Skill 负责“先追问，再研究”
- browser 负责读网页来源
- read/write/edit 负责把结果写回工作区
- memory 负责沉淀研究画像
- cron 放到后续持续跟踪阶段再上

## 8. 为什么建议单独 agent

推荐不要直接拿 `main` agent 跑这条线，而是单独建一个类似：

```text
industry-research-assistant
```

原因：

- 行业研究和个人待办不是一类上下文
- 后面这条线会有自己独立的来源偏好、输出格式和 cron
- 单独 agent 更方便调规则、看日志、做实验迭代

## 9. 第一版工作区建议

这条线后面建议逐步沉淀下面这些文件：

```text
MEMORY.md
memory/research-profile.md
memory/daily/YYYY-MM-DD.md
sources/priority-sources.md
outputs/YYYY-MM-DD.md
```

但第一轮实验不需要一次全配满。

第一轮先保证有：

- 研究画像
- 当天输出
- 来源清单

就够了。

## 10. 示例案例怎么放

这条通用实验线后面可以自然长出多个具体案例。

第一批最适合的案例就是：

- 调味品行业每日趋势
- OpenClaw / Agent 产品赛道跟踪

这些案例都不应该替代通用框架，而应该挂在通用框架下面。

## 11. 第一轮成功标准

第一轮实验通过，至少要满足：

- 能在飞书里先把研究简报问清楚
- 能把模糊需求收敛成一份清晰任务卡
- 能基于这份任务卡跑出第一版研究结果
- 能把结果写回工作区文件
- 后面能自然接到持续跟踪
